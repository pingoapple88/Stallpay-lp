# 派工單 13：Stallpay-lp 代理登入按鈕調整

## §0 任務目標
1. **移除 LP 右上角「成為夥伴」按鈕**：避免直客在首頁被分散注意力。
2. **底部新增「代理登入」按鈕**：將原本的「成為夥伴」功能移至頁面最底端，並更名為「代理登入」，以符合直客攤主登入頁的設計需求。
3. **維持 UX/UI 簡單易用**：確保改動不破壞原有的版面平衡，不造成用戶混淆。

## §1 範圍與邊界
- **影響檔案**：
  - `index.html`
  - `partner.html` (若有連帶影響)
- **排除項目**：不修改現有代理商入口網站 (`dealer-portal`) 的功能，僅調整 LP 的跳轉按鈕。

## §2 實作規格
### 2.1 移除右上角按鈕
- 在 `index.html` 的 `<nav>` 區塊中，找到 `<button class="btn btn-dealer btn-sm" id="dealerBtn">成為夥伴</button>` 並將其移除。
- 移除相關的 CSS 樣式（若該樣式僅被此按鈕使用）。

### 2.2 底部新增「代理登入」按鈕
- 在 `index.html` 的 `<footer>` 區塊（約 1008 行之後）或其上方，新增一個「代理登入」的連結或按鈕。
- 點擊「代理登入」時，觸發原本 `openDealer()` 函式的邏輯，或直接導向代理商登入頁面（視原有邏輯而定）。
- 確保樣式與底部文字（如「老闆登入」）一致，保持低調但可點擊。

### 2.3 檢查 `partner.html`
- 確認 `partner.html` 中是否有類似的按鈕需要同步調整，以保持全站一致性。

## §3 驗收條件
1. LP 首頁（`index.html`）右上角不再顯示「成為夥伴」按鈕。
2. LP 首頁最底端出現「代理登入」連結/按鈕。
3. 點擊底端的「代理登入」能正確開啟代理商入口（或跳出原有的 modal）。
4. 整體網頁視覺無跑版，在手機版（LINE In-App Browser）下顯示正常。

## §4 主權守則與八律對照
- [x] **平台中立**：僅為純前端 HTML/CSS 調整，不依賴特定平台 API。
- [x] **白標可擴展**：UI 調整符合直客與經銷雙軌並行的設計邏輯。
